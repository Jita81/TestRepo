📱 Viewport Testing

✅ Test Cases

Test 1: Login Form is Usable on Mobile Devices (320px width) Pending

Given: I am viewing the login page on a mobile device with 320px width

When: The page loads

Then: Verify the following:

  • All form elements are visible and properly sized
  • Input fields are at least 44px tall
  • Text is readable without zooming (minimum 16px)
  • Submit button is easily tappable
  • No horizontal scrolling occurs
  • Spacing prevents accidental taps

Test 2: Registration Form Adapts to Tablet Screen (768px) Pending

Given: I am viewing the registration page on a tablet device

When: The page loads in portrait mode (768px width)

Then: Verify the following:

  • Form is centered with appropriate margins
  • Form fields stack vertically with comfortable spacing
  • Password requirements list is clearly visible
  • All elements are proportionally sized
  • Touch targets are appropriately sized

Test 3: Dashboard Navigation Collapses on Small Screens Pending

Given: I am logged in and viewing the dashboard on a mobile device

When: The screen width is below 768px

Then: Verify the following:

  • Main navigation collapses into hamburger menu
  • Tapping menu icon reveals navigation options
  • Menu can be dismissed by tapping outside or close button
  • Menu items are easily selectable with touch
  • Logout option is always accessible

Test 4: Desktop View Utilizes Available Space (1920px) Pending

Given: I am viewing any authentication page on a large desktop screen

When: The screen width is 1920px or greater

Then: Verify the following:

  • Forms are constrained to maximum readable width (600-800px)
  • Content is centered horizontally
  • White space is used effectively
  • Text line length remains readable
  • All elements maintain proper proportions

Test 5: Landscape Orientation on Mobile Works Correctly Pending

Given: I am viewing a form on a mobile device

When: I rotate the device to landscape orientation

Then: Verify the following:

  • Layout adjusts appropriately
  • Form remains fully visible without excessive scrolling
  • Submit buttons remain accessible
  • Viewport meta tag prevents unwanted zooming

Test 6: Touch and Click Interactions Work Across Devices Pending

Given: I am using either touch or mouse input

When: I interact with buttons, links, and form controls

Then: Verify the following:

  • Touch targets are at least 44x44px on mobile
  • Hover states work on desktop
  • Active/pressed states visible on mobile
  • Focus indicators visible for keyboard navigation
  • No functionality exclusive to one input method

Test 7: Images and Icons Scale Appropriately Pending

Given: The authentication pages include logos or icons

When: I view pages at different screen sizes

Then: Verify the following:

  • SVG icons scale without quality loss
  • Logos are appropriately sized (larger on desktop, smaller on mobile)
  • Images use responsive techniques (srcset, picture element)
  • All images have appropriate alt text
